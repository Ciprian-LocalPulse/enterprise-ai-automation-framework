## Submitting a Support Request

### Via Email

Send to [CONTACT@LOCALPULSE.PRO](mailto:CONTACT@LOCALPULSE.PRO) with the following subject format:

```
[SUPPORT] [License Tier] Module XX — Brief Description
```

**Include in your email:**
- License tier and purchase reference
- Blueprint version (e.g., v3.2)
- Module or template number affected
- Detailed description of the issue
- Steps to reproduce (for technical issues)
- Expected vs. actual outcome
- Environment details (tools, platforms, integrations)

### Via GitHub Issues

Use the appropriate issue template:
- **Bug Report** — For errors in documentation, templates, or frameworks
- **Feature Request** — For enhancement suggestions (authorized contributors only)

**Do not open GitHub Issues for:**
- Security vulnerabilities (use [SECURITY.md](./SECURITY.md))
- Licensing questions (use email)
- General business strategy questions (book a consultation)
