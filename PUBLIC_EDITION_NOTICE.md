# Public Edition Notice

Enterprise AI Automation Framework is published here as a Public Edition for enterprise evaluation, public documentation, portfolio review, architecture discussion, and commercial discovery.

## Purpose of the Public Edition

The Public Edition is intended to communicate:

- enterprise automation and governance product direction
- architecture-level thinking
- public governance posture
- interface and documentation quality
- procurement-safe positioning
- compliance-aware framing without exposing private implementation details

## What the Public Edition May Include

The public repository may include:

- README and product positioning
- static landing page or public interface preview
- sanitized architecture descriptions
- non-operational examples
- governance documents
- safe support and security reporting instructions
- high-level commercial evaluation language

## What the Public Edition Does Not Include

The Public Edition does not include and should not be used to infer:

- production automation workflows
- orchestration logic or execution playbooks
- private AI prompts, system prompts, tool instructions, or evaluation traces
- customer-specific implementations
- private integrations with CRMs, ERPs, payment systems, ticketing systems, cloud accounts, or internal databases
- access-control rules, approval chains, escalation paths, or data transformation policies
- private compliance evidence, SOC 2 artifacts, ISO 27001 evidence, GDPR records, or audit workpapers
- API keys, tokens, service-account files, credentials, secrets, or environment variables
- customer data, client environments, vendor records, or internal process maps
- commercial implementation details, acquisition terms, pricing, or buyer-specific requirements

## Private Enterprise Edition

The Private Enterprise Edition, if made available, is separate from this public repository and may only be accessed under a written agreement.

Private Enterprise Edition discussions should remain private and should not be opened as public GitHub issues when they involve implementation detail, pricing, buyer requirements, compliance evidence, customer environment assumptions, automation design, prompt logic, or confidential evaluation criteria.

## Responsible Use

Enterprise AI Automation Framework is positioned for lawful enterprise automation, security architecture, governance, compliance-readiness, and public evaluation purposes. Public materials must not be used to support unauthorized access, credential misuse, unsafe automation, data misuse, compliance misrepresentation, or activity outside a lawful and authorized context.

## Public Communication Standard

Issues, pull requests, discussions, and references to this repository should remain calm, factual, enterprise-safe, and security-conscious. Do not publish sensitive findings or private implementation assumptions in public channels.
