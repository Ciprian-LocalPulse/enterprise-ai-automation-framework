# Contributing Policy

Thank you for your interest in Enterprise AI Automation Framework. This repository is maintained as a controlled Public Edition and enterprise evaluation surface.

## Contribution Model

External code contributions are not accepted unless explicitly approved by the repository owner in advance.

This restriction protects the Public Edition and prevents accidental disclosure of private enterprise implementation details, commercial materials, compliance evidence, automation logic, prompts, customer-specific workflows, or security-sensitive information.

## Contributions That May Be Considered

The maintainer may selectively consider non-sensitive contributions such as:

- typo fixes or documentation clarity improvements
- governance file improvements
- accessibility notes for public documentation
- issue reports about broken links or inaccurate public claims
- safe suggestions for public repository hygiene

Please open an issue before preparing any pull request. Unapproved pull requests may be closed without review.

## Contributions That Will Not Be Accepted

Do not submit, request, or describe:

- production automation workflows, orchestration logic, or execution playbooks
- private AI prompts, system prompts, tool instructions, evaluation traces, or internal prompt libraries
- API keys, tokens, service-account files, credentials, environment variables, or secrets
- customer data, client environments, vendor records, process maps, or internal runbooks
- private compliance evidence, SOC 2 artifacts, ISO 27001 evidence, GDPR records, or audit workpapers
- private integrations with CRMs, ERPs, payment systems, ticketing systems, cloud accounts, or internal databases
- access-control rules, approval logic, escalation paths, data transformation rules, or production policies
- commercial implementation details, acquisition materials, pricing models, or buyer-specific requirements
- changes that enable unauthorized automation, data misuse, credential misuse, compliance misrepresentation, or unsafe operational activity

## Public Edition Boundary

The Public Edition communicates safe product direction, architecture-level thinking, governance maturity, interface quality, and commercialization readiness. It is not the Private Enterprise Edition and does not accept contributions that attempt to recreate private implementation details.

## Pull Request Expectations

If the maintainer explicitly invites a contribution:

1. Keep the change narrow and documentation-focused unless otherwise approved.
2. Do not modify product logic, design, assets, README sales positioning, or private implementation details unless specifically requested by the maintainer.
3. Avoid adding dependencies, scripts, data collection, workflow automation, prompt logic, integration behavior, telemetry, or executable behavior.
4. Keep language calm, enterprise-safe, procurement-safe, and legally careful.
5. Confirm that no secrets, credentials, customer references, private prompts, compliance evidence, or private methods are included.

## Issues

Public issues should be limited to non-sensitive repository concerns. Do not open public issues containing vulnerability details, prompt details, workflow logic, customer information, private architecture, implementation-sensitive details, or commercial negotiation material. Use SECURITY.md for security reporting.

## License and Ownership

The public repository materials are governed by the applicable license files in this repository. Private Enterprise Edition materials, commercial implementation details, confidential workpapers, and customer-specific materials are not included in the Public Edition.

## Maintainer Discretion

The maintainer may close issues or pull requests that are out of scope, unsafe, duplicative, commercially sensitive, legally sensitive, or inconsistent with the purpose of this repository.
